Key Cross-Module Contracts

session_id
Generated by Ingestion Orchestrator. Carried through raw tables → allocation_grain → reconciliation_results → final.allocation_result → every export row (second-to-last column).
billing_period YYYY-MM
Enforced at IAM, Billing, ERP validators. Derived by Billing Period Deriver via LEFT(date,7). Relied on by IAM Resolver, Check 2, and Check 3 as join key.
failed_tenant_id
Set by Identity Broken Record Builder (= original tenant_id). NULL in Type A and capacity_idle rows. Used by Customer Data Aggregator to fire the Risk FLAG.
unallocated_type
capacity_idle = reserved, no job ran. identity_broken = job ran, IAM failed. Distinct at grain level, never blended. Drives Region pills and Export Verifier Check 4.
write_result
Set by Approved Result Writer → State Store in one atomic transaction with application_state = APPROVED. Read by Export Gate Enforcer. Survives State Machine restarts.
Closure Rule
SUM(gpu_hours per pool per day) = reserved_gpu_hours. Idle forced as a first-class Type B row before any aggregation. Blending is architecturally impossible.
